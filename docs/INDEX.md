# harvest-gui — Documentation

## Structure

| Directory | Purpose |
|-----------|---------|
| `reference/` | API docs, configuration, architecture |
| `design/draft/` | Design documents in progress |
| `design/approved/` | Approved designs, ready for implementation |
| `audits/` | Code reviews, quality audits |
| `sessions/` | Session notes (YYYY-MM/SESSION_YYYY-MM-DD.md) |

## Conventions

- **UPPERCASE.md** for strategic/reference docs
- **kebab-case.md** for guides and supporting docs
- **Max 30KB per file** — split larger documents
- **Never delete** — move to `archive/` instead
